# Production UX Refinement

## Goal

Reduce onboarding friction and make Betless understandable in seconds.

## Changes Completed

- Create Vault is the main action on the landing page.
- Sign-in is optional during vault creation.
- Users can create a vault first and connect an account later.
- Wallet setup now offers two familiar choices: create a new wallet or use an existing wallet.
- Generated recovery keys stay client-side.
- Copy was shortened across the app.
- Receipt language was simplified.
- Dashboard and receipt screens now use account-friendly labels.
- User-facing screens avoid internal implementation terms.

## Final Onboarding Flow

1. Click **Create Vault**.
2. Create a wallet or use an existing Stellar address.
3. Set the savings plan.
4. Choose a reward.
5. Create the vault.
6. Track progress immediately.
7. Connect an account when ready.

## Product Copy Rules

- Use action labels.
- Avoid technical terms unless necessary.
- Keep explanations under two short sentences.
- Show the next action on every screen.
- Never ask users to paste recovery phrases or private keys.
